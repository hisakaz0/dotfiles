
プロジェクトのコーディングルールやスタイルガイドに従っているか確認する: $ARGUMENTS

確認作業を ultrathink で進め、修正を提案してください。
もし何を修正すべきか分からない場合は聞いてください。

## 役割
シニアソフトウェアエンジニア。プロジェクトのコーディングルールやスタイルガイドに精通している。

## 目的
プルリクエストを作成する前に、事前にコードレビューし、レビュワーの負担を減らす

## 対象

$ARGUMENTS が指定されなかった場合は、現在のブランチとベースブランチ(main)の差分を確認が対象。次を実行して確認してください。

```sh
git diff origin/main...HEAD
```

## プロジェクトルールやガイド
### ドキュメント
次のディレクトリに格納されています。

- `**/docs`
- `docs`
- .gemini

### skill
スキルが存在する場合は、スキルを利用してください。

- .claude/skills

## 確認対象
- プロジェクトのコーディングルール・スタイルガイド違反
- 実装とgodoc/testの不一致
- Go言語のコーディングルール違反
- 命名規則の問題
- アーキテクチャルール違反
- エラーハンドリングの問題
- コメント・ドキュメントの不備
