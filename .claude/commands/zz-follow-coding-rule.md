
プロジェクトのコーディングルールとスタイルガイドに従って、現在のブランチの変更を確認・修正してください。

役割：
シニアソフトウェアエンジニアとして、Pull Request前のコードレビューと修正を実行

手順：
1. 現在のブランチとベースブランチ(main)の差分を確認
  * 差分の確認には `git diff origin/main...HEAD` を実行する
2. 次の項目を確認する
  * プロジェクトのコーディングルール・スタイルガイドにマッチしているかどうか（CLAUDE.mdに記載）
  * 差分の実装とgodoc/testが対応しているかどうか
3. 明らかな問題がある箇所を特定・修正
4. 判断が難しい箇所は指摘と修正方針を報告

確認対象：
- Go言語のコーディングルール違反
- 命名規則の問題
- アーキテクチャルール違反
- エラーハンドリングの問題
- コメント・ドキュメントの不備
