引数のキーワード「$ARGUMENTS」で過去のClaudeCodeセッションを横断検索し、関連度上位5件の **セッションID** と **直近の会話の要約** を、`/resume <session id>` 形式でコピペ即実行できる形で出力してください。

セッション数は2000件超(数百MB)になるため、**速度優先**で以下の手順を厳守してください。

## 引数のパース

`$ARGUMENTS` を以下のオプションに分解する。

- `--days N` : 検索対象を直近 N 日に制限（既定: `7`）。`--days 0` で全期間検索
- `--cwd <path>` : `cwd` フィールドが指定パスを含むセッションのみ対象
- それ以外のトークンを連結したものを **検索キーワード** とする（複数語可）

## 検索手順

セッションファイルは `~/.claude/projects/<エンコード済みパス>/<セッションID>.jsonl` に保存されています。各行は1メッセージのJSONで、`type` が `user` / `assistant` のものが会話本体です。

### 1. mtimeで対象ファイルを先に絞る（最重要）

`grep -r` を全件に対して走らせると遅いので、**先に `find` で期間絞り込み**してからファイル名を `grep` に渡す。

```bash
# 既定: 直近7日
find ~/.claude/projects -name "*.jsonl" -type f -mtime -7 -size +0c -print0 \
  | xargs -0 grep -lI -- "<キーワード>" 2>/dev/null
```

- `--days 0` 指定時は `-mtime` を付けず全期間を対象にする
- `--days N` (N>0) 指定時は `-mtime -N` を使う
- `-print0` / `xargs -0` でスペース・特殊文字を安全に扱う
- `-I` でバイナリスキップ、`-l` でファイル名のみ出力
- `--` でハイフン始まりのキーワードを安全に渡す

ヒットが0件かつ `--days 0` 以外が指定されている場合は、以下のフォールバックを **1回だけ** 行う:

1. `--days 30` 相当で再検索
2. それでも0件なら `--days 0` 相当で全期間を再検索

各フォールバック実行時はユーザにその旨を1行表示する（例: `直近7日では見つからず、30日に範囲を拡大して再検索します`）。

### 2. キーワード関連度でランク付け

**最終的な5件は日付順ではなく「キーワードとの関連度」で並べる。**

各マッチファイルに対して、以下のスコアを計算する:

```bash
# 会話本体(type=user/assistant)に絞ってヒット数をカウント
grep -c -- "<キーワード>" <ファイル>                # 全体ヒット数(score_total)
grep -E '"type":"(user|assistant)"' <ファイル> \
  | grep -c -- "<キーワード>"                       # 会話本体ヒット数(score_body)
```

スコアリング:

- **score_body**: 会話本体ヒット数（`type=user`/`type=assistant` の行内）
- **score_total**: 全体ヒット数（メタデータ含む）
- **mtime**: ファイル更新時刻（タイブレーク用）

ソート順は **`score_body` 降順 → `score_total` 降順 → `mtime` 降順** とする。

```bash
# 各ファイルについて "body_hits total_hits mtime path" を出力してソート
for f in <マッチしたファイル群>; do
  body=$(grep -E '"type":"(user|assistant)"' "$f" | grep -c -- "<キーワード>")
  total=$(grep -c -- "<キーワード>" "$f")
  mtime=$(stat -f "%m" "$f")
  printf "%d %d %d %s\n" "$body" "$total" "$mtime" "$f"
done | sort -k1,1nr -k2,2nr -k3,3nr | head -15
```

`score_body == 0` のファイルはメタデータのみのヒット（`hookEventName` 等のノイズ）なので、他に候補がある場合は除外する。全件 0 の場合のみ `score_total` ベースで残す。

候補が5件以下ならその数で次へ進む。

### 3. 並列サブエージェントで要約

**メインコンテキストを汚染しないため**、関連度上位5件の要約は **Agent ツール（subagent_type=general-purpose）で並列実行** する。1メッセージ内で5件まとめて起動すること。

各サブエージェントへのプロンプトは以下の通り:

```
以下のClaudeCodeセッションファイルを末尾付近のみ読んで要約してください。

ファイル: <絶対パス>

抽出する情報:
- sessionId: ファイル名から `.jsonl` を除いた値
- cwd: cwd フィールド
- gitBranch: gitBranch フィールド
- 末尾近くの type=user の最新メッセージ
- 末尾近くの type=assistant の最新メッセージの主旨

ファイルが大きい場合は Read の offset で末尾だけ読む（先頭から読まない）。

最大200語で、以下のフォーマット1件分を返してください:

<セッションID>
- cwd: <cwd>
- branch: <gitBranch>
- 最終更新: <YYYY-MM-DD HH:MM> (statで取得)
- 要約: <2〜3文で直近の会話の要点を日本語で>
```

並列起動例:

> Agent #1〜#5 を1つのメッセージにまとめて呼び出すこと。逐次呼び出しは禁止。

## 出力フォーマット

サブエージェントから返ってきた要約を **関連度順（score_body 降順）** に並べ、各エントリの先頭を **`/resume <セッションID>` 形式** にする。コードブロックで囲み、コピペでそのまま実行できるようにすること。日付順ではないことに注意。

```
1. /resume <セッションID>
   - 要約: <2〜3文で直近の会話の要点を日本語で>
   - 関連度: 会話本体ヒット <body>件 / 全体 <total>件
   - cwd: <cwd>
   - branch: <gitBranch>
   - 最終更新: <YYYY-MM-DD HH:MM>

2. /resume <セッションID>
   ...

3. /resume <セッションID>
   ...

4. /resume <セッションID>
   ...

5. /resume <セッションID>
   ...
```

冒頭に1行で **検索条件のサマリ** を表示する（例: `キーワード: "syncCreate" / 期間: 直近7日 / 候補: 8件 → 関連度上位5件を要約`）。

## 注意

- 出力は最大5件。マッチがそれ以下なら見つかった分だけ出力する。
- マッチが0件かつフォールバックでも見つからない場合は「『<キーワード>』にマッチするセッションは見つかりませんでした。」と表示して終了する。
- 要約は会話の **直近** の文脈に絞る（中盤や冒頭ではなく、末尾付近のやり取り）。
- メインコンテキストで `.jsonl` 本体を Read しない（巨大なため）。Read はサブエージェント側で行う。
- `find` の前に `cd` しない。`~/.claude/projects` を絶対パスで指定する。
