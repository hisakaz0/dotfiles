
現在の実装を次のコマンドで確認してください。

```sh
git diff origin/main...HEAD 
```

実装に対して次のユニットテストを洗い出してください。

* 実装とテストの仕様があっておらず、テストの変更が必要なユニットテスト
* 実装だけがあり、テストの追加が必要なユニットテスト
* テストだけががあり、テストの削除が必要なユニットテスト

テストの変更や追加に関しては、テストケースを ultrathink で考えてください。
追加・削除・変更するユニットテストとそのテストパターンがリスト形式でまとめたら、そのまま進めてください。

もしテストの変更や追加で分からないことがあれば、止めてください。
