PRを分割するべきか検討する

## ルール
* extend thinking を有効化する
* ultrathink で対応する
* git repositoryが見つからない場合は即時終了する

## 確認箇所
* `git df`を実行し、修正内容を確認してください。
* `git l` を実行し、コミット内容を確認してください。

## PR分割の観点
* リファクタリングとバグ修正など、変更目的が1つではない
* `*_gen.go` や `*_test.go` 以外の変更が 500 行を超える
    * `e2e/**/*_adminc` の変更は無視でOK

内容を確認した上で、1つのPRにすべきか、複数のPRに分割すべきか答えをだしてください。
結論は、Aが60%, Bが40% など数値で出してください。
